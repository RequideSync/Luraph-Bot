# 📦 LuraPh Bot 使い方ガイド

---

## 🚀 起動方法

```bash
python bot.py
```

> Windows の場合、管理者権限が必要です。自動で昇格プロンプトが表示されます。

起動すると以下が確認できます：

```
起動完了 | Bot: LuraPh#XXXX | 在庫: 5 / 金額: 220円
```

---

## 🤖 ボットのステータス表示

起動後、15秒ごとに自動でステータスが切り替わります。

| 表示例 | 説明 |
|---|---|
| `在庫: 5個 \| 220円` | 現在の在庫と金額 |
| `Buy LuraPh With PayPay!` | 購入案内 |
| `Only 220円!` | 金額アピール |
| `💴 220円 で購入できます` | 日本語案内 |
| `📦 残り在庫: 5` | 在庫数 |

> 在庫が **0** になると、ステータスが 🔴 **Do Not Disturb（赤）** に自動変更されます。

## 👤 ユーザー向けコマンド

### `/register` — PayPay で購入する

PayPay リンクを使って LuraPh を購入します。

1. `/register` を実行するとフォームが開きます
2. **お名前** と **PayPay リンク** を入力して送信
3. 管理者が承認すると **クレジット +1** が付与されます
4. 承認・拒否の結果は **DM** で通知されます

> ⚠️ 在庫切れの場合は購入できません。  
> ⚠️ 金額が設定金額と一致しない場合はエラーになります。

---

### `/credit` — クレジット・在庫を確認する

自分のクレジット残高と現在の在庫・購入金額を確認できます。

```
👤 あなたのクレジット: 2
📦 現在の在庫: 3
💴 購入金額: 220円
```

---

### `/luraph [ファイル]` — オーダーを作成する

クレジットを **1消費** して、処理依頼ファイルを送信します。

1. `/luraph` にファイルを添付して実行
2. 管理サーバーに専用チャンネルが自動作成されます
3. 管理者がファイルを処理後、`L!luraph` コマンドで返却されます

> ❌ クレジットが **0** の場合は実行できません。  
> ⚠️ チャンネル作成に失敗した場合、クレジットは自動で**返還**されます。

## 🔧 管理者向けコマンド（プレフィックス: `L!`）

> ⚠️ 以下のコマンドは **管理者ロール** を持つユーザーのみ実行できます。

---

### `L!SetAmount [金額]` — 購入金額を設定する

PayPay で受け取る金額を設定します。

```
L!SetAmount 500
→ ✅ PayPay認証金額を 500円 に設定しました。
```

---

### `L!AddStock [数量]` — 在庫を追加する

在庫数を増やします。

```
L!AddStock 10
→ ✅ 在庫を 10 追加しました。現在の在庫: 15
```

---

### `L!SetStock [数量]` — 在庫を直接設定する

在庫数を指定した数に上書きします。

```
L!SetStock 5
→ ✅ 在庫を 5 に設定しました。
```

---

### `L!AddCredit [@ユーザー or ID] [数量]` — クレジットを付与する

指定ユーザーにクレジットを手動付与します。

```
L!AddCredit @user 3
→ ✅ @user に 3 クレジットを付与しました。（残高: 3）
```

---

### `L!RemoveCredit [@ユーザー or ID] [数量]` — クレジットを削除する

指定ユーザーのクレジットを削除します（0未満にはなりません）。

```
L!RemoveCredit @user 1
→ ✅ @user から 1 クレジットを削除しました。（残高: 2）
```

---

### `L!CheckCredit [@ユーザー or ID]` — クレジットを確認する

指定ユーザーのクレジット残高を確認します。

```
L!CheckCredit @user
→ 👤 @user のクレジット: 2
```

---

### `L!Status` — ボットのステータスを確認する

現在の在庫・金額設定を Embed で確認します。

```
📊 ボットステータス
在庫: 5
設定金額: 220円
```

---

### `L!luraph [@ユーザー or ID] + ファイル添付` — ファイルを返却する

処理済みファイルをユーザーの DM に送信し、チャンネルのログ保存後に自動削除します。

```
L!luraph @user  ← ファイルを添付して送信
```

**処理の流れ：**

1. ✅ 指定ユーザーの DM にファイルを送信
2. 📁 チャンネルのログを `Log/` フォルダに保存（最大500件）
3. 🕐 5秒後にチャンネルを自動削除

> ⚠️ ユーザーの DM が閉じている場合は送信に失敗します（チャンネルは削除されません）。

---

## 📁 ファイル構成

```
bot.py              # ボット本体
credits.json        # ユーザーごとのクレジット管理
settings.json       # 金額・在庫設定
bot.log             # 実行ログ（自動生成）
Log/
  └─ order-xxx_YYYYMMDD_HHMMSS.txt  # オーダーチャンネルのログ
```

---

## ⚙️ settings.json の構造

```json
{
    "target_amount": 220,
    "stock": 5
}
```

| キー | 説明 |
|---|---|
| `target_amount` | PayPay で受け取る金額（円） |
| `stock` | 現在の在庫数 |

---

## ❓ よくあるエラー

| エラーメッセージ | 原因 | 対処法 |
|---|---|---|
| 金額が一致しません | PayPay リンクの金額と設定金額が違う | `L!SetAmount` で金額を確認 |
| このリンクは既に使用済みです | PayPay リンクが使用済み | 未使用のリンクを使う |
| 在庫切れです | 在庫が 0 | `L!AddStock` で在庫を追加 |
| クレジット不足 | クレジットが 0 | `/register` で購入 |
| DM送信失敗 | ユーザーの DM が閉じている | ユーザーに DM を開くよう伝える |
